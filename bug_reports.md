# Баг-репорт 1

|Заголовок | [bug][web][prod] Не отображается статус потери сети при отключении интернета во время дозвона через веб-версию |
|:----------|:----------------------------------------------------------------------------------------------------------------|
| Описание | При отключении интернет-соединения во время дозвона интерфейс не отображает статус потери сети. Через некоторое время дозвон автоматически завершается без уведомления пользователя. В консоли браузера отображаются сетевые ошибки. |
| Предусловия | **1.** Пользователь X и пользователь Y авторизованы  и имеют стабильное интернет соединение.<br>**2.** Пользователь Y не должен принимать звонок |
| Шаги воспроизведения| **1.** Пользователь X инициирует звонок пользователю Y.<br>**2.** Пока идет дозвон необходимо отключить интернет на стороне пользователя X. |
| Текущий результат | **1.** Интерфейс не сообщает о проблеме с сетью - Отсутсвует уведомление о статусе сети - **"Подключение/Нет сети"**.<br>**2.** Через некоторое время дозвон автоматически прервыется без каких либо уведомлений о проблеме.<br>**3.** В консоли отображаются сетевые ошибки ![](/screenshots/network_errors[bug1].jpg) |
| Ожидаемый результат | **1.** Появление уведомления о статусе сети **"Подключение/Нет сети"** <br>**2.** Прекращение подключения по истечении времени ожидания |
| Окружение | 1. VK WEB<br>2. Билд **v5.199**<br>3. Браузер Google Chrome Версия: 145.0.7632.160 (официальная сборка) (arm64) |
| Тестовые устройства | Macbook Pro 16 M1 Pro macOS Tahoe 26.1 (25B78)|
| Priority | Medium |
| Severity | Minor |
| Логи | Если бы я работал в VK, то прикрепил бы ссылку на логи из Kibana или Sentry |

# Баг-репорт 2

|Заголовок | [bug][web][prod] Несинхронизированное завершение звонка при потере соединения  |
|:----------|:--------------------------------------------------------------------------------|
| Описание | При потере интернет-соединения у одного из участников активного звонка состояние звонка становится несинхронизированным между клиентами. У одного пользователя звонок завершается, тогда как у второго продолжает отображаться попытка подключения. |
| Предусловия | Пользователь X и пользователь Y авторизованы  и имеют стабильное интернет соединение. |
| Шаги воспроизведения| Пользователь X инициирует звонок пользователю Y. |
| Текущий результат | **1.** После отключения интернета во время звонка у пользователя X происходит попытка соединения у обоих пользователей.<br>**2.** Через некоторое время у пользователя Y завершается попытка соединения и прекращается звонок (только на его стороне).<br>**3.** У пользователя X в это продолжается попытка соединения и если в этот моент восстановить интернет, то попытка продолжается некоторое время, а после завершается звонок. (то есть значительно позже, чем у пользователя Y)<br>**4.** В результате пользователи видят разные состояния звонка. |
| Ожидаемый результат | При потере интернет-соединения у одного из участников:<br> **1.** Cистема должна корректно обработать разрыв соединения.<br>**2.** Cостояние звонка должно быть синхронизировано между участниками.<br>**3.** После истечения времени ожидания звонок должен завершиться у обоих пользователей одновременно.<br>**4.** Интерфейс должен отобразить одинаковый статус завершения звонка |
| Окружение | 1. VK WEB<br>2. Билд **v5.199**<br>3. Браузер Google Chrome Версия: 145.0.7632.160 (официальная сборка) (arm64) |
| Тестовые устройства | Macbook Pro 16 M1 Pro macOS Tahoe 26.1 (25B78)|
| Priority | Medium |
| Severity | Major |
| Логи | Если бы я работал в VK, то прикрепил бы ссылку на логи из Kibana или Sentry |
